# Token Redesign — Design Spec

## Goal

Replace the current flat text-in-div piece tiles with circular SVG tokens that look like coins/medals — rank number centered, piece name curved along the top arc in a blackletter font, with a player-chosen color from a preset palette on the setup screen.

## Architecture

Frontend-only. No backend changes. Tokens are inline SVGs generated by `game.js`. The Cruz Quaste blackletter font is self-hosted as a `.woff2` file. Player color choice is stored in `localStorage` and read at render time.

## Token Design

Each piece on the board is an inline SVG rendered inside the existing `.board-cell` div.

### SVG structure

- **Outer circle:** Filled with the player's chosen color (yours) or red (`#8b4444`, enemy). Stroke border 2-3px darker shade.
- **Inner ring:** Subtle decorative circle (1px, low opacity white) for the coin/medal feel.
- **Center symbol:** Bold, large text — rank numbers `1`–`9` for Marshal through Scout, italic `S` for Spy (rank 10), `💣` for Bomb, `🚩` for Flag, `?` for unrevealed enemy.
- **Curved name:** Piece name (e.g. "Marshal", "Scout", "Spy") curved along the top arc of the circle using `<textPath>` on an SVG arc path. Font: Cruz Quaste (blackletter). Only shown for your own pieces and revealed enemy pieces (not for `?` unknowns).

### Token sizing

- Token SVG occupies ~70% of the cell dimensions, centered in the cell.
- The cell's wood-colored background (`#c9a876`) remains visible around the token.
- Board cells no longer change their background color based on piece presence — they stay wood-colored. Only the token circle itself carries the player color.

## Color Picker

### Setup screen

- A row of 8 preset color swatches displayed on the setup screen, above or alongside the formation controls.
- Colors (curated for contrast against the wood board and red enemy):
  1. Forest Green `#4a7a4a` (default, current green)
  2. Navy Blue `#3a5a8a`
  3. Royal Purple `#6a4a8a`
  4. Teal `#3a7a7a`
  5. Gold `#8a7a3a`
  6. Crimson `#8a3a4a`
  7. Slate `#5a6a7a`
  8. Bronze `#8a6a3a`
- Each swatch is a clickable circle. Selected swatch gets a gold highlight ring.
- Default selection: Forest Green (index 0).

### Storage

- On selection, writes to `localStorage` key `stratego:<roomCode>:color` with the hex value.
- `game.js` reads this key at render time. If absent, defaults to Forest Green.
- Enemy pieces always render as red (`#8b4444`) regardless of what the opponent chose — you only control your own token color.

### Darker stroke

- The token's border stroke is computed as a darker version of the chosen color. Implementation: reduce each RGB component by ~30% (e.g. `#4a7a4a` → `#2a5a2a`).

## Font

- **Font:** Cruz Quaste by Måns Grebäck. Free for personal use (non-commercial license) — this project is non-commercial.
- **Hosting:** Self-hosted at `web/fonts/CruzQuaste.woff2`. Loaded via `@font-face` in `styles.css`.
- **Fallback:** `serif` — if the font fails to load, the curved name still renders in a serif font.
- **Usage:** Only for the curved `<textPath>` name on tokens (and optionally graveyard labels). Not used for general UI text.
- The `.woff2` file needs to be converted from the `.ttf` download. Use an online converter or `woff2_compress`.

## Graveyard Consistency

- Graveyard filled slots should use the same player color (not hardcoded green/red) for "your" pieces.
- Enemy graveyard slots stay red.
- The graveyard slots remain simple divs (not full SVGs) — just update the background color to match the player's chosen color for `.filled-mine` slots.

## Board Cell Changes

- `.board-cell.mine` and `.board-cell.enemy` classes are removed — cells no longer change background color based on piece presence.
- Cells always show wood background. The SVG token provides all the piece visual.
- `.board-cell.selected` outline (gold ring) stays — it highlights the cell, which frames the token nicely.
- Hover states stay on cells (slightly darker wood), not on tokens.

## Files Changed

- **Create:** `web/fonts/CruzQuaste.woff2` — self-hosted font file
- **Modify:** `web/css/styles.css` — `@font-face` for Cruz Quaste, remove `.board-cell.mine`/`.board-cell.enemy` background colors, add `.piece-token` sizing, color picker swatch styles, update `.graveyard-slot.filled-mine` to use CSS variable for player color
- **Modify:** `web/js/game.js` — `renderBoard()` generates inline SVG tokens instead of text content, reads player color from localStorage, `renderGraveyards()` uses player color for filled-mine slots
- **Modify:** `web/js/setup.js` — color picker rendering + click handler + localStorage write
- **Modify:** `web/setup.html` — color picker section in the setup layout

## Non-Goals

- No animation on token placement or movement.
- No per-piece custom colors (one color for your whole army).
- No opponent color choice visibility (you don't see what they picked).
- No changes to the move log text format.
- No changes to the setup grid piece placement (those stay as the current chips, not SVG tokens).
